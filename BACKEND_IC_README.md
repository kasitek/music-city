# Music City IC Backend (Motoko)

This folder contains an Internet Computer canister written in Motoko that implements the core backend for Music City.

- Canister name: `music_city_backend`
- Config file: `dfx.json`
- Source: `src/music_city_backend/main.mo`
- Types: `src/music_city_backend/types.mo`

## Prerequisites

- Install the DFINITY SDK (dfx): https://internetcomputer.org/docs/current/developer-docs/getting-started/install/
- Node.js LTS (for frontend integration later)

## Quick Start (Local)

1. Start a local replica in a terminal:

```bash
# from the repo root
dfx start --clean --background
```

2. Deploy the canister:

```bash
dfx deploy
```

3. Get the canister id:

```bash
dfx canister id music_city_backend
```

4. Call example methods from the CLI (Candid):

```bash
# Register an artist (runs as anonymous principal by default)
dfx canister call music_city_backend registerUser '("Alex", variant { artist }, "Bio", "LA", vec {"Pop"}, "/img.png", null)'

# Create a track (after registration)
dfx canister call music_city_backend createTrack '("Electric Dreams", "3:24", "Pop", "/cover.jpg", "/audio.mp3", 10, "2025-08-01", "Desc")'

# List tracks
dfx canister call music_city_backend listTracks '()'

# Stream track id=1
dfx canister call music_city_backend streamTrack '(1)'

# Mint an NFT
dfx canister call music_city_backend mintNFT '("Electric Dreams - LE", "/art.png", 1000, variant { rare }, "Artwork")'

# Purchase NFT id=1
dfx canister call music_city_backend purchaseNFT '(1)'
```

## What it Implements

- Users: register, get/update profile, balances stored per principal
- Tracks: create/list/get, stream increments plays and credits artist with a royalty
- Tipping: transfer from caller to artist with balance checks
- NFTs: mint (artist only), list, purchase with 10% platform fee retained
- Transactions: recorded and queryable by caller

All state is stable across canister upgrades.
